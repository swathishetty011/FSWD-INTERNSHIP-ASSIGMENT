Client-Server Architecture

Client-Server Architecture is a network model where the client (user device or browser) requests services or data from a server. The server processes the request and sends the required information back to the client.

Components

  • Client: User device or web browser (Chrome, Firefox, Edge)
  • Server: Computer that stores website data
  • Internet: Network that connects client and server

What Happens When You Open a Website?

  1. User enters the website URL in the browser.
  2. The browser sends a DNS request to find the website's IP address.
  3. The DNS server returns the IP address.
  4. The browser sends an HTTP/HTTPS request to the web server.
  5. The server processes the request and sends website files.
  6. The browser receives HTML, CSS, and JavaScript files.
  7. The browser renders the webpage and displays it to the user.

Conclusion

Client-server architecture is the basic working model of the internet. When a user opens a website, the browser communicates with the server, requests data, and displays the webpage on the screen.
